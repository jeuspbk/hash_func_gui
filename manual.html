1. 소개

이 도구는 키의 타입과 구조를 입력하면 그에 맞는 C 언어 해시 함수를 만들어 주는 데스크톱 프로그램입니다. Python 표준 라이브러리(Tkinter)만으로 동작합니다.

세 가지 모드를 제공합니다.

모드용도충돌
범용(General)임의의 키를 잘 분산시키는 일반 해시 (해시 테이블용)분포는 균일하나 충돌은 발생
Perfect미리 정해진 고정 키 목록에 대한 충돌 없는 해시0 (보장)
구조분석(Spec)문자열 키의 구조(세그먼트)로부터 해시 자동 생성가능하면 0, 불가하면 일반 해시

2. 설치 및 실행

별도 설치가 필요 없습니다. Python 3.10 이상(Tkinter 포함)만 있으면 됩니다.

명령줄에서

python main.py             # GUI 실행
python main.py --selftest  # GUI 없이 내부 자가검증

Windows에서 더블클릭으로

파일동작
run.bat콘솔 창과 함께 실행. 오류가 나면 메시지를 유지합니다.
run_silent.batpythonw로 콘솔 없이 GUI만 실행합니다.

배치 파일은 자신이 있는 폴더를 작업 폴더로 자동 설정하므로 어디서 실행해도 됩니다.

5. 모드 1 — 범용(General)

키 집합을 미리 몰라도 되는 일반 해시입니다. 키 타입에 따라 알고리즘을 고릅니다.

정수 키

알고리즘특징
splitmix강한 비트 혼합(권장 기본)
wangThomas Wang 정수 해시
knuth곱셈(피보나치) 해시

정수 인자 폭(32/64bit)도 선택합니다.

문자열 키

알고리즘특징
fnv1aFNV-1a(권장 기본)
djb2Bernstein 해시
sdbmSDBM 해시

옵션: 대소문자 무시, NUL 종료(해제 시 (ptr, len) 시그니처).

생성 예 (문자열 / FNV-1a / 32bit)

uint32_t myhash(const char *s) {
    uint64_t h = 2166136261ULL;
    while (*s) { uint64_t c = (unsigned char)*s++; h ^= c; h *= 16777619ULL; }
    return (uint32_t)h;
}

6. 모드 2 — Perfect(완전 해시)

미리 정해진 고정 키 목록에 대해 충돌이 전혀 없는 해시를 만듭니다. 키워드 테이블, 명령어 디스패치 등에 적합합니다.

사용법

  1. 키 타입(문자열/정수)을 고릅니다.
  2. 키 목록 칸에 키를 한 줄에 하나씩 입력합니다.
  3. 옵션을 선택하고 결과를 확인합니다.
옵션설명
최소 완전 해시(minimal)키 n개를 정확히 0..n-1로 매핑(가장 조밀)
검증 테이블/lookup 방출원본 키 배열과 <함수>_lookup()을 함께 생성 — 미지의 키를 -1로 구분
내부적으로 CHD-lite(hash-and-displace) 알고리즘으로 충돌 0을 탐색합니다. 문자열 1000개도 수 ms 내에 생성됩니다.

7. 모드 3 — 구조분석(Spec)

문자열 키의 구조(세그먼트별 위치·길이·문자종류)를 입력하면 그에 맞는 해시를 자동으로 만듭니다. 기본 목표는 완전 해시(충돌 0)입니다.

7.1 세그먼트 편집기

각 행이 키의 한 구간을 나타냅니다.

칸의미
시작구간이 시작하는 바이트 오프셋(0부터)
길이N(고정) / a~b(가변 범위) / *(가변, 끝까지) — 가변은 마지막 세그먼트만
문자종류아래 프리셋 선택 또는 직접 입력

+ 세그먼트 추가로 행을 늘리고, 각 행의 ✕로 삭제합니다. 추가 시 시작 오프셋은 자동 추정됩니다.

문자종류 프리셋

라벨토큰크기
영문자(대소)[a~z][A~Z]52
소문자 / 대문자[a~z] / [A~Z]26
숫자[0~9]10
영숫자[a~z][A~Z][0~9]62
16진수(소) / (대)[0~9][a~f] / [0~9][A~F]16
사용자 정의: 문자종류 칸은 직접 입력도 됩니다. 범위 토큰 [x~y]와 리터럴을 조합하세요. 예: [0~9][a~f], [!~/](기호), [가~힣]

7.2 보조 입력

항목설명
예상 키 개수byte-hash로 갈 때 테이블 크기와 시뮬레이션 표본 수를 정하는 데 사용
패킹 한계 도메인패킹 도메인이 이 값 이하이면 밀집 완전 해시를 선택. 기본 4194304
테스트 입력샘플 키를 넣으면 그 키의 해시값을 즉시 표시 (정상 동작 확인용)

7.3 전략 자동 선택

“패킹 도메인” = 각 세그먼트 문자종류 크기의 곱(= 구별 가능한 키의 총 개수)입니다. 이 크기에 따라 전략이 자동으로 정해집니다.

조건전략반환충돌
도메인 ≤ 패킹 한계 (그리고 ≤ 2³²)packed-dense최소 완전 해시uint32_t, 0..도메인-10
도메인 < 2⁶⁴packed-wide완전 해시(희소)uint64_t 고유 인덱스0
도메인 ≥ 2⁶⁴ 또는 가변 길이byte-hash일반 해시uint32_t (% M)가능 ⚠
packed-index(혼합 진법)란? 숫자 구간은 10진법, 영문자 구간은 26진법… 처럼 각 세그먼트를 진법 값으로 바꿔 자리값으로 결합합니다. 그 결과 모든 유효 키가 [0, 도메인)의 서로 다른 정수로 매핑됩니다(전단사 = 완전 해시). 숫자/16진수만으로 된 키는 입력값이 곧 출력값이 됩니다.
왜 항상 작은 완전 해시는 안 되나요? 비둘기집 원리상, 키공간이 D개인데 테이블을 D보다 작게 만들면 반드시 충돌이 생깁니다. 따라서 구조만으로 만드는 완전 해시는 테이블 크기 = 도메인입니다. 도메인이 너무 크면(또는 가변 길이라 무한이면) 완전 해시가 불가능해 일반 해시로 폴백하며 그 사실을 경고로 알립니다. 작은 테이블 + 충돌 0이 필요하면 실제 키 목록으로 Perfect 모드를 사용하세요.

생성 예 — packed-index (숫자2 + 소문자3, 도메인 1,757,600)

uint32_t idx(const char *s) {
    const unsigned char *p = (const unsigned char *)s;
    uint32_t idx = 0;
    idx = idx * 10u + idx_r0[p[0]];   /* 숫자: 10진법 */
    idx = idx * 10u + idx_r0[p[1]];
    idx = idx * 26u + idx_r1[p[2]];   /* 소문자: 26진법 */
    idx = idx * 26u + idx_r1[p[3]];
    idx = idx * 26u + idx_r1[p[4]];
    return idx;                        /* 0..1757599, 충돌 0 */
}

가변 길이 키

마지막 세그먼트 길이를 * 또는 a~b로 두면 가변 길이 키를 다룹니다. 이 경우 생성 코드는 strlen 기반 경계 검사를 하므로 스펙보다 짧은 키에도 메모리 오류 없이 안전합니다. (단, 가변 길이는 완전 해시가 아니라 일반 해시입니다.)

8. 유틸리티 — 다음 소수

설정 패널 맨 아래의 유틸리티: 다음 소수 칸에 숫자를 입력하면 그보다 큰 첫 소수를 즉시 표시합니다. 해시 테이블 크기를 소수로 고를 때 편리합니다.

입력출력
10001009
4194304 (2²²)4194319
1000000000000000000 (10¹⁸)1000000000000000003

결정적 Miller-Rabin 소수 판정을 사용해 큰 수도 정확합니다.

9. 생성 코드 사용법

  • 미리보기의 헤더(.h) / 소스(.c) 탭에서 결과를 확인합니다.
  • .h 저장 / .c 저장으로 파일로 내보내거나, 클립보드 복사(.c)로 복사합니다.
  • 저장한 <함수>.c를 프로젝트에 추가하고, 사용하는 곳에서 <함수>.h를 include합니다.

C에서 사용 예

#include "myhash.h"

unsigned slot = myhash("hello");        /* 범용/구조분석 */
/* 해시 테이블 인덱스로 바로 사용하거나, 필요하면 % 테이블크기 */
생성된 .c 파일 상단 주석에는 선택된 전략·엔트로피·충돌 수 등 판단 근거가 기록됩니다.

10. 핵심 개념

용어뜻
완전 해시(perfect hash)주어진 키들이 서로 다른 값으로 매핑되어 충돌이 없는 해시
최소 완전 해시(minimal)키 n개를 빈틈없이 0..n-1로 매핑
패킹 도메인구조 스펙으로 구별 가능한 키의 총 개수(문자종류 크기의 곱)
혼합 진법(mixed-radix)세그먼트마다 다른 진법으로 자리값을 매겨 고유 정수로 변환하는 방법
엔트로피키가 담을 수 있는 정보량(비트). byte-hash가 처리할 바이트를 고르는 기준
비둘기집 원리칸보다 물건이 많으면 한 칸에 둘 이상 → 테이블이 키공간보다 작으면 충돌 불가피

11. 자주 묻는 질문

Q. 구조분석에서 해시 출력이 입력 문자열과 다릅니다. 버그인가요?

아닙니다. 해시는 입력을 고유한 인덱스 숫자로 바꾸는 것이 목적입니다. 숫자/16진수만으로 된 키는 입력값과 같아지지만, 영문자가 섞이면 압축된 인덱스가 나옵니다. 테스트 입력란으로 같은 키가 항상 같은 값(충돌 0)을 내는지 확인할 수 있습니다.

Q. byte-hash로 떨어졌는데 충돌 없이 만들고 싶어요.

도메인이 너무 크거나(≥2⁶⁴) 가변 길이라 구조만으로는 완전 해시가 불가능한 경우입니다. ① 고정 길이/작은 스펙으로 줄이거나, ② 실제 키 목록이 있으면 Perfect 모드를 사용하세요.

Q. 실행이 안 됩니다.

Python 3.10 이상(Tkinter 포함)이 설치되어 PATH에 있는지 확인하세요. run.bat은 오류 시 메시지를 남깁니다. 명령줄에서 python main.py --selftest로 핵심 동작을 점검할 수 있습니다.
